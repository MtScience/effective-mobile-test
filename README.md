# effective-mobile-test

Тестовое задание для Effective Mobile.

## Запуск

Для запуска необходимы Docker и Docker Compose.

1. Создать файл `.env` с переменными
   ```
   POSTGRES_DB="<your db name>"
   POSTGRES_USER="<your db user>"
   POSTGRES_PASSWORD="<youur db password>"
   POSTGRES_PORT=<db port>
   HTTP_ADDRESS="<http address>"
   ```
   
   Рекомендуемые значения для `POSTGRES_PORT` и `HTTP_ADDRESS` — `5432` и `":8080"` соответственнос.

2. Запустить команду
   ```shell
   docker compose up --build
   ```

3. После запуска Swagger будет доступен по адресу `<http_address>/swagger`. Если в `.env` было указано значение
   `HTTP_ADDRESS=":8080"`, то адрес будет `localhost:8080/swagger`.

## Комментарии по заданию

Проект реализован на Go 1.26.1 с использованием библиотек `gorilla`, `pgx` и `google/uuid`.

В задании реализованы стандартные операции добавления, удаления, модификации и получения (по ID и списком) подписок,
а также операция подсчёта стоимости подписок за заданный период. Поскольку в задании не описано, что именно
подразумевается под фильтрацией по ID пользователя и названию подписки, были реализованы три варианта: фильтрация
только по ID пользователя, только по названию подписки и по обоим полям (связь через `AND`). Такое решение связано с
тем, что в реальной ситуации необходимость фильтрации по обоим полям сразу маловероятна, поскольку у одного
пользователя обычно не бывает больше одной подписки на один сервис. Более того, в реальном проекте соответствующие
поля с большой вероятностью составляли бы первичный ключ таблицы (иначе говоря, комбинации из ID пользователей и
подписок были бы уникальны). Тем не менее, поскольку в настоящем проекте такого ограничения нет, возможность фильтрации
по обоим поля сразу также реализована.
